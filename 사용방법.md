# 📺 Mini Video Streaming Service
FastAPI와 MinIO(S3 Compatible Storage)를 이용한 초간단 비디오 업로드 및 스트리밍 환경 구축 가이드입니다.

### 🏗️ 서비스 구조
* **FastAPI**: 비즈니스 로직 및 API 엔드포인트 (Port: `8000`)
* **MinIO**: 비디오 파일 저장소 (Port: `9000`, 웹 콘솔: `9001`)

---

### 🐳 도커(Docker) 주요 명령어

프로젝트 루트 폴더( `docker-compose.yml`이 있는 곳)에서 실행하세요.

| 명령어 | 설명 |
| :--- | :--- |
| `docker-compose up -d --build` | **(중요)** 코드를 수정한 후 이미지를 새로 빌드하고 실행 |
| `docker-compose up -d` | 컨테이너를 백그라운드에서 실행 |
| `docker-compose logs -f` | 실행 중인 서버의 로그를 실시간으로 확인 |
| `docker-compose down` | 모든 컨테이너 중지 및 네트워크 삭제 |
| `docker-compose down -v` | 저장된 데이터(볼륨)까지 모두 삭제 (초기화) |

### 🚀 API 엔드포인트 사용 방법 (Curl & Browser)

#### 1. 비디오 파일 업로드
내 컴퓨터의 로컬 파일을 서버로 전송합니다.
```bash
curl -X 'POST' \
  'http://localhost:8000/upload/' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@내비디오파일.mp4'
```

#### 2. 업로드된 비디오 목록 확인
현재 스토리지에 어떤 파일들이 있는지 확인합니다.
```bash
# 1. 원본 파일 목록 확인 (videos 버킷)
curl -X 'GET' 'http://localhost:8000/videos/origin'

# 2. 변환된 파일 목록 확인 (transcoded 버킷)
curl -X 'GET' 'http://localhost:8000/videos/transcoded'
```

#### 3. 비디오 스트리밍 URL 가져오기
브라우저에서 재생 가능한 임시 보안 URL(Presigned URL)을 생성합니다.
```bash
# sample.mp4 자리에 실제 파일명을 입력하세요.
# 720p 영상을 가져오고 싶을 때 (파일명 규칙에 주의!)
curl -X 'GET' 'http://localhost:8000/video/720p/sample_720p.mp4'
```

#### 4. 대화형 API 문서 (Swagger UI)
웹 브라우저에서 직접 테스트하고 싶을 때 사용하세요.
* 주소: [http://localhost:8000/docs](http://localhost:8000/docs)

#### 5. 스토리지 관리자 화면 (MinIO Console)
저장된 파일을 GUI로 직접 관리(삭제, 다운로드 등)하고 싶을 때 접속하세요.
* 주소: [http://localhost:9001](http://localhost:9001)
* ID/PW: `minioadmin` / `minioadmin`

### ⚠️ 주의사항 (Troubleshooting)
* **DNS 에러 (`NXDOMAIN`) 발생 시**: API가 반환한 URL에 `http://storage:9000`이 포함되어 있다면, 코드에서 이를 `http://localhost:9000`으로 바꿔주는 `replace()` 로직이 적용되었는지 확인하고 `--build`로 재실행하세요.
* **파일 경로**: `curl` 명령어 실행 시 `@` 뒤에 오는 파일명은 현재 터미널이 위치한 경로에 실제로 존재하는 파일이어야 합니다.
